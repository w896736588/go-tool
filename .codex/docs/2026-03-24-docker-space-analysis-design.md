# Docker 空间分析与日志清理设计

## 目标

在 `/Docker` 页面增加两个能力，均基于当前页面选择的 SSH 环境执行：

1. Docker 空间分析
2. 清理容器日志

## 范围

- 空间分析统计当前环境机器上的全部容器，不限制在当前 Compose 配置列表内。
- 展示每个容器的基础信息、容器空间占用、日志占用。
- 返回汇总信息，便于页面快速展示总容器数、总日志占用和总空间占用。
- 清理日志执行 `truncate -s 0 /var/lib/docker/containers/*/*-json.log`，由后端远程执行。

## 后端方案

- 在 `internal/pkg/p_shell/command.go` 增加 Docker 空间分析命令和清理日志命令。
- 在 `internal/app/dtool/controller/docker.go` 增加：
  - Docker 空间分析接口
  - Docker 日志清理接口
  - 空间分析结果解析函数
- 空间分析命令输出统一为制表符分隔，字段包括：
  - 容器 ID
  - 容器名
  - 镜像
  - 状态
  - 日志路径
  - 日志字节数
  - 可写层字节数
  - 根文件系统字节数
- 控制器在解析后补充：
  - 人类可读大小
  - 汇总统计
- 为空间解析增加单元测试，覆盖：
  - 正常解析
  - 空日志文件
  - 非法行过滤
  - 汇总计算

## 前端方案

- 在 `web/src/utils/base/compose.js` 增加两个接口调用方法。
- 在 `web/src/components/Docker.vue` 顶部工具栏增加两个按钮：
  - `空间分析`
  - `清理容器日志`
- 空间分析通过弹窗展示：
  - 顶部汇总卡片
  - 容器列表表格
- 清理容器日志增加二次确认，成功后自动刷新空间分析结果。

## 风险与处理

- `docker inspect --size` 依赖远端 Docker 可用，失败时透传错误给前端。
- 某些容器日志路径可能为空，按 `0` 处理。
- 页面不在前端直接执行危险命令，所有实际操作都走后端。
