# 首页任务清单 AI 工作日报设计

## 目标

在首页任务清单上方增加“一键生成工作日报”能力。

用户提前在设置页配置好日报模型和日报提示词后，点击首页按钮即可由后端读取当前所有未归档任务，调用 LLM 生成 Markdown 日报，并自动写入记忆库，同时打上 `工作日报` 标签。

## 背景与约束

- 首页任务清单已有未归档任务列表接口 `/api/HomeTaskList`。
- 项目已有 AI 模型配置体系，可通过 `tbl_ai_model` 与 `InfoCrawlChatByModel` 复用 LLM 调用链。
- 项目已有记忆保存能力，可通过 `MemoryFragmentSave` 对记忆库写入并触发同步。
- 需要避免首页出现额外复杂表单，因此模型与提示词改为在设置页单独维护。
- 只要未归档任务为空、记忆未配置、模型未配置等不确定性会影响功能语义时，直接在后端返回明确错误。

## 用户流程

### 设置页

在现有“记忆配置”页新增“工作日报 AI”配置区：

- 日报模型：仅允许选择 `llm` 类型模型
- 日报提示词：多行文本

保存后写入全局配置，立即生效。

### 首页

首页任务清单顶部工具栏新增 `AI 生成工作日报` 按钮：

- 点击后不弹窗
- 前端直接调用新接口
- 生成中显示 loading，避免重复点击
- 成功后提示“工作日报已生成并保存到记忆”
- 失败后提示后端返回的明确错误

## 数据来源

工作日报只基于当前未归档任务生成。

后端在生成时实时查询未归档任务，避免直接使用前端缓存数据。每条任务至少包含以下字段：

- 任务名称
- 当前状态
- 开始时间
- 最后操作时间
- 备注

如果时间字段为空，统一显示为 `-`，避免提示词上下文出现歧义。

## AI 生成规则

### 配置项

新增两项全局配置：

- `home_task_daily_report_model_id`
- `home_task_daily_report_prompt`

默认提示词由后端提供常量，避免配置为空时出现不可控输出。

### Prompt 结构

调用模型时采用：

- system prompt：固定约束为“你是工作日报整理助手，输出中文 Markdown，只基于提供任务，不要编造”
- user prompt：由“用户自定义提示词 + 当前任务快照”拼接而成

任务快照建议采用清晰的结构化文本，例如：

```text
日期：2026-03-24
任务总数：3

任务 1
- 名称：任务A
- 状态：开发中
- 开始时间：2026-03-24
- 最后操作时间：2026-03-24 14:20:00
- 备注：补充接口联调
```

## 记忆写入规则

AI 返回结果后，由后端直接保存到记忆库：

- 标题：`工作日报 YYYY-MM-DD`
- 内容：AI 生成后的 Markdown
- 标签：固定包含 `工作日报`

保存成功后，沿用现有记忆同步机制触发 `ScheduleSync`。

## 错误处理

以下场景直接返回错误，不保存空记忆：

- 记忆库未配置
- 日报模型未配置
- 日报模型不存在或不是 `llm`
- 日报提示词为空且默认提示词不可用
- 当前没有未归档任务
- AI 调用失败
- 记忆保存失败

## 复用点

### 前端

- 复用首页 `Home.vue` 的任务清单工具栏
- 复用 `git_set.js` 的记忆配置读写入口
- 复用 `ai_set.js` 的 LLM 模型列表接口

### 后端

- 复用 `SetMemoryConfigGet` / `SetMemoryConfigSave` 的配置读取保存模式
- 复用 `InfoCrawlAiModelInfo` 与 `InfoCrawlChatByModel`
- 复用 `memoryDBOrResponse` 与 `MemoryFragmentSave` 的记忆保存路径

## 测试重点

### 后端单测

- 默认日报提示词回退
- 日报 user prompt 任务快照拼装
- 任务为空时拒绝生成
- 模型非 `llm` 时拒绝生成
- 生成标题格式符合 `工作日报 YYYY-MM-DD`

### 前端回归

- 设置页能加载并保存日报模型与提示词
- 首页按钮在点击时进入 loading，并在回调后恢复
- 生成成功与失败提示符合预期

## 实施边界

本次只做手动触发生成，不做定时任务、不做日报历史列表、不做生成前预览。
