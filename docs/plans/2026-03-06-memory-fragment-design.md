# 记忆片段设计稿

Date: 2026-03-06

## 概述
新增一个独立主菜单“记忆片段”，用于保存 Markdown 格式的知识片段。该模块支持自由标签、按标签过滤、关键词检索、自然语言检索、内容历史记录、假删除，以及右侧多文档 Tab 打开体验。首页不是系统 `/Dashboard`，而是记忆片段模块自己的欢迎页，并作为默认 Tab 存在。

本模块的目标是“更强的检索能力”，不是 AI 问答系统。系统只负责把相关片段找出来，不负责基于片段生成答案。

## 目标
- 在左侧提供片段列表，支持快速浏览、创建、删除和切换。
- 在右侧提供欢迎页、搜索区、多标签筛选区和多文档 Tab 工作区。
- 每个片段支持保存 Markdown 标题与正文，并维护自由标签。
- 支持关键词检索与自然语言检索两种方式。
- 任何内容变更都保留历史记录。
- 删除使用假删除，避免误删后无法恢复。
- 为后续检索能力预留异步索引链路，避免保存时阻塞用户操作。

## 非目标
- 不改造当前通用 `Markdown.vue` 的抽屉文档能力。
- 不做复杂权限模型、分享链接、协同编辑。
- 不做 AI 问答、Agent 编排、自动摘要、自动改写。
- 不在当前 Go 主项目中额外引入 Bun 运行时。

## 推荐方案
采用独立“记忆片段”业务模块，检索层采用“SQLite 主存储 + FTS5 全文检索 + sqlite-vec 向量检索 + 异步索引任务”的组合方案。

后端新增独立的 memory 相关表与接口；前端新增独立页面、路由和 API 封装。现有 `Markdown` 模块继续保留，用于当前 Git、变量、接口等场景，不与“记忆片段”混用。

这样可以避免把“欢迎页 + 多文档 Tab + 高级检索 + 假删除 + 历史快照”这些业务语义强塞进通用 Markdown 模块里，后续维护风险更低。

## 技术取舍
### 采用
- `SQLite`：主存储
- `FTS5`：全文检索
- `sqlite-vec`：向量检索
- `goqite` 经 `internal/taskmanager` 封装：异步索引任务

### 不采用
- `Bun + SQLite`
  原因：当前项目主后端是 Go，额外引入 Bun 会增加双运行时与双部署复杂度，对当前目标收益不高。

- `CloudWeGo Eino / Eino ADK`
  原因：当前只做检索，不做问答、Agent、工具编排，暂不需要。

## 备选方案
### 方案一：直接扩展 `tbl_markdown` 与 `Markdown.vue`
- 优点：开发表面上更快。
- 缺点：现有组件是“单文档编辑器”模式，不适合多 Tab 工作区；检索、历史、删除语义也会影响已有调用方。

### 方案二：前端独立页面，后端仍复用 `tbl_markdown`
- 优点：比独立新表改动更小。
- 缺点：数据语义混杂，后续 FTS、向量索引、异步任务都会污染通用 Markdown 数据。

## 数据模型
建议新增五部分数据结构：

### `tbl_memory_fragment`
主表，保存片段基础信息。

字段建议：
- `id`
- `title`：标题
- `content`：Markdown 正文
- `content_text`：从 Markdown 提取的纯文本，供索引任务或调试使用
- `is_deleted`：是否假删除，`0/1`
- `index_status`：索引状态，`pending/success/failed`
- `index_version`：当前索引版本号，用于避免旧任务覆盖新索引
- `create_time`
- `update_time`

说明：
- `content_text` 是辅助字段，不作为最终检索主入口。
- 真正的检索依赖 FTS5 与向量索引。

### `tbl_memory_fragment_tag`
标签关联表，一个片段可绑定多个自由标签。

字段建议：
- `id`
- `fragment_id`
- `tag_name`
- `create_time`
- `update_time`

约束建议：
- 同一片段下标签去重
- 标签写入前统一 `trim`
- 空标签直接忽略

### `tbl_memory_fragment_history`
历史表，记录每次保存时的变更快照。

字段建议：
- `id`
- `fragment_id`
- `title_old`
- `title_new`
- `content_old`
- `content_new`
- `tags_old`
- `tags_new`
- `change_desc`
- `create_time`
- `update_time`

说明：
- `tags_old/tags_new` 优先存 JSON 字符串。
- `change_desc` 用于历史列表简述，沿用当前项目已有 diff 摘要思路。

### `tbl_memory_fragment_fts`
FTS5 虚表，用于关键词检索。

索引内容建议：
- 标题
- 正文纯文本
- 标签文本拼接结果

用途：
- 精确词命中
- 多词组合命中
- 前缀和短语查询
- 与标签过滤叠加

### `tbl_memory_fragment_vec`
向量索引表，用于自然语言检索与相似片段召回。

字段建议：
- `fragment_id`
- `embedding`
- `model_name`
- `model_version`
- `create_time`
- `update_time`

说明：
- 具体向量存储方式依赖 `sqlite-vec` 的接入方式。
- 该表只服务检索，不承载问答逻辑。

## 索引任务模型
建议通过 `goqite` 经 `internal/taskmanager` 封装后处理异步索引任务。

任务职责：
- 提取 Markdown 纯文本
- 更新 FTS5 索引
- 生成 embedding
- 更新向量索引
- 更新片段索引状态

任务特点：
- 保存主数据与检索索引解耦
- 减少前端等待时间
- 为后续批量重建索引提供统一入口

## 检索设计
### 检索模式
支持三种使用方式：

1. 关键词检索
- 用户输入短词、标签词、标题词
- 优先走 `FTS5`
- 返回更精确的文本匹配结果

2. 自然语言检索
- 用户输入完整问题或口语句子
- 优先走向量检索
- 返回语义相近的片段

3. 混合检索
- 同时查询 `FTS5` 与向量索引
- 结果合并、去重、加权排序
- 作为默认推荐方案

### 关键词检索规则
- 输入框按空格切词
- 逻辑仍保持“全部命中”
- 每个词需要在标题、正文或标签中命中
- 标签筛选与关键词检索叠加

说明：
- 在 FTS5 层可通过构造 `AND` 查询表达式实现“全部命中”语义。

### 自然语言检索规则
- 将用户整句输入转成 embedding
- 用向量相似度检索 TopK 片段
- 默认排除已删除片段
- 若同时存在标签筛选，则在向量召回结果上再过滤标签

### 混合排序建议
排序分数可由以下部分组成：
- `fts_score`
- `vec_score`
- 更新时间衰减权重
- 标签精确命中加权

本期只需预留这个策略，不要求一次性把排序做复杂。

## 页面交互设计
### 整体布局
主页面采用左右布局：
- 左侧：所有片段列表，固定宽度
- 右侧：顶部搜索区 + 标签筛选区 + Tab 工作区

整体视觉沿用当前项目卡片化、浅色、绿色系风格，不另起新设计语言。

### 左侧片段列表
左侧展示所有未删除片段，列表项包含：
- 标题
- 标签摘要
- 更新时间
- 删除按钮

左侧顶部提供：
- “新建片段”按钮
- 可选的列表内快速过滤输入框

列表交互：
- 点击片段：在右侧打开对应 Tab；若已打开，则切换到该 Tab
- 当前激活片段在左侧高亮
- 删除时弹出确认框，执行假删除

### 右侧欢迎页
首次进入模块时，默认打开“首页”Tab。

欢迎页内容建议：
- 搜索输入框
- 已选标签区
- 常用标签区或全部标签区
- 最近更新片段列表
- 最近检索命中片段区
- “新建片段”快捷入口

欢迎页本身不承载编辑器，只负责检索、导航和入口汇总。

### 右侧搜索区
顶部常驻一个搜索输入框：
- 支持关键词短语输入
- 支持自然语言整句输入
- 提供清空按钮
- 输入后触发检索结果刷新

建议增加一个轻量检索模式切换：
- `混合`
- `关键词`
- `自然语言`

默认模式建议为 `混合`。

标签筛选区展示：
- 所有标签或按使用频率排序的标签
- 点击标签后进入选中状态
- 再次点击可取消

### Tab 工作区
Tab 行固定显示在右侧搜索区下方。

规则：
- 默认存在“首页”Tab，且不可关闭
- 打开片段后新增对应 Tab
- 同一片段只允许打开一个 Tab
- 支持切换 Tab
- 支持关闭除首页外的片段 Tab
- 关闭当前 Tab 后自动切换到最近一个仍打开的 Tab；若无则回到首页

### 片段详情 Tab
片段 Tab 页面内容包含：
- 标题输入框
- 标签输入框
- Markdown 编辑器
- 保存按钮
- 历史记录按钮
- 删除按钮
- 索引状态展示

索引状态展示建议：
- `待索引`
- `索引成功`
- `索引失败`

标签输入交互建议：
- 使用可编辑的标签输入框
- 以回车、逗号或空格提交单个标签
- 自动去重
- 支持删除已有标签

保存规则：
- 手动点击保存
- 保存成功后立即更新主数据
- 检索索引异步更新，不阻塞保存结果反馈

未保存状态：
- 若标题、正文、标签有任一修改，Tab 标题追加 `*`
- 切换片段或关闭 Tab 前先尝试保存，或弹出确认；本期建议采用“切换前自动保存一次”

### 历史记录交互
点击“历史记录”后，打开弹窗展示当前片段历史列表。

每条历史显示：
- 变更时间
- 变更摘要
- 操作：查看对比

查看对比时：
- 对标题、标签、正文变化分别展示
- 正文沿用现有 Markdown diff 组件风格

### 删除交互
- 片段删除仅更新 `is_deleted = 1`
- 已删除片段不出现在默认列表中
- 若删除的是当前已打开 Tab，对应 Tab 自动关闭；若是当前激活 Tab，则切回首页

## 数据流
### 首次进入模块
1. 路由进入 `/MemoryFragment`
2. 拉取片段列表、标签列表、最近更新片段
3. 默认激活“首页”Tab

### 打开片段
1. 用户点击左侧片段
2. 若 Tab 已存在，则切换
3. 若不存在，则拉取详情并打开新 Tab

### 保存片段
1. 收集标题、正文、标签
2. 后端更新主表
3. 后端重建该片段标签
4. 若有变化则写入历史表
5. 后端将该片段标记为 `pending`
6. 后端投递异步索引任务
7. 前端刷新左侧列表、欢迎页最近更新与标签区

### 异步索引
1. Worker 拉取待处理任务
2. 提取 Markdown 纯文本
3. 更新 FTS5 索引
4. 生成 embedding
5. 更新向量索引
6. 更新索引状态为成功或失败

### 执行检索
1. 用户输入检索条件
2. 前端传递检索模式、关键字、标签筛选
3. 后端按模式执行 FTS、向量或混合查询
4. 结果合并排序后返回

### 删除片段
1. 用户确认删除
2. 后端执行假删除
3. 前端刷新左侧列表与右侧打开状态
4. 删除后的片段在检索结果中默认不可见

## 风险与处理
### 风险一：向量生成耗时
通过异步任务解耦，不阻塞保存链路。

### 风险二：旧索引覆盖新内容
通过 `index_version` 或更新时间校验，防止旧任务写回新片段。

### 风险三：检索结果排序不稳定
本期先提供基础混合排序，后续再逐步调参。

### 风险四：向量模型替换
向量表中记录模型名和版本，便于后续重建索引。

- 可创建、编辑、查看 Markdown 片段
- 可维护自由标签
- 支持基于标签与关键词的全文检索
- 支持自然语言检索
- 支持混合检索
- 保存后可查看历史记录
- 删除为假删除
- 右侧支持首页 Tab 与多个片段 Tab 共存
