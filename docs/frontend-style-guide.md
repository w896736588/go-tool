# 前端页面与弹窗风格规范

后续所有前端页面与弹窗，默认沿用当前 `Supervisor / Docker / 接口开发(Api)` 已落地的视觉语言，除非用户明确要求改风格或项目已有设计系统要求。

## 1. 总体视觉
- 使用卡片化布局：白底、浅边框、圆角、轻阴影。
- 主色调使用当前项目绿色系（避免引入全新主色）。
- 页面背景避免强对比纯色，保持低干扰。

## 2. 组件规范
- 顶部操作区：统一为标题区 + 控制行（筛选/按钮/搜索）。
- 表格：统一表头底色、hover 效果、行状态色。
- 输入框/选择器：统一圆角与 focus 边框风格。
- 主按钮/次按钮：颜色、边框、hover 行为保持一致。
- 按钮形态默认使用“图标 + 文字”（如：`<el-icon />` + 文案），保证可读性与识别效率；仅在空间极小或纯装饰场景下使用纯图标按钮。

## 3. 弹窗规范
- 所有 `el-dialog` 默认使用统一样式（圆角、边框、阴影、头尾分割、统一内边距）。
- 遮罩层透明度与色调保持一致，不为单页单独重写冲突样式。

## 4. 响应式
- 桌面端优先保证信息密度与操作效率。
- 小屏下自动纵向堆叠，避免横向溢出和关键按钮不可见。

## 5. 实施要求
- 优先复用已有样式实现。
- 参考文件：
  - `web/src/components/Supervisor.vue`
  - `web/src/components/Docker.vue`
  - `web/src/components/Api.vue`
  - `web/src/css/reset_element.css`
- 新样式优先写语义化 class，减少内联 style。
- 若新增样式与本规则冲突，先调整为一致再提交。
